A time I was Blocked on a Simple Problem

The Challenge: using .join to take individual strings from an array and return one string

I confidently jumped into this one as I had previously researched this and had used it in one of our previous exercises. So here I go happily writing the code and to my surprise the test fails. Hmm. Ok. I check the spelling of the parameter. No problems. I check the right brackets were used. No problems either. At this point I'm starting to scratch my head, so I did what most would do...I deleted it and started again

This time when I wrote the code I changed the names of the parameters (thinking that this would actually change the outcome haha) but the same red fail message came up. I also realised at this point, I hadn't had a break in a while so did that and when I came back it dawned on me to maybe check what the fail message says and check the test file and boom! I had put a space in between the quote marks when the test required no space...sheesh

I felt a bit frustrated that it was simple but then started to laugh because it was so simple! What I learnt here was that I just needed to slow down and realise that there's help around (in this case it was in the form of the error message and test file) and to actually use the help that's available


A Time I Elegantly Solved a Problem

The Challenge: superFlipFlop

This one gave me a lot of satisfaction as it had a Loop element to it and I had put a lot of time into understanding them and how to write the code. Which of course at first there were many iterations of is i greater than or less than and making sure all 3 types of brackets were in the right places

It took a few goes and it passed the test. Yay! Great! But looking at my code, it seemed like I was doing 1 or 2 steps too many so dove into trying to refactor it

During this step, I found that I'd actually defined an extra Object that was doing nothing as I never used it during the For Loop, so got rid of that. I also went back over Dot Notation and changed the loops output to one line instead of defining a variable to equal the output and then using that variable to be pushed into the empty array

A Time I was Reluctant to ask for Help

Naturally, I find it hard to ask for help at the best of times. I enjoy the challenge of not knowing and using what little knowledge I have of something to figure it out...but there's always a time when you do need help and that time was earlier this week

I was trying to do my best and work my way through the JS Kata exercises, but I did need to reach out to a colleague of mine to help with the understanding of Object Manipulation. All of the Googling and Tutorials couldn't help me to understand the how, what and why, so I gave in and phoned a friend

After receiving the help I needed, turns out I just need someone to explain it to me like i'm 5, I was away and finished the exercise with minimal fuss. The lesson here is to not be afraid and to not be so proud to not ask for help
